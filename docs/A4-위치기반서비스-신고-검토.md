# A4. 위치기반서비스사업 신고 대상 여부 검토

> 검토일: 2026-07-17 · 결론: **현재 구조에서는 신고 대상 아님** (아래 "재검토 트리거" 발생 시 재판단)

## 1. 판단 기준 (위치정보법)

위치기반서비스사업 신고(방송통신위원회) 대상 여부는 **"사업자가 이용자 개인의 위치정보(GPS 좌표·Wi-Fi AP·기지국 Cell 등)를 수집·이용·제공하는가"**로 판단한다.

- 개인 위치정보를 서버로 전송받아 이용하면 **DB에 저장하지 않아도** 신고 대상.
- 위치정보가 **단말기 안에서만 처리되고 사업자 서버로 전송되지 않으면** 신고 대상 아님.
- 가게·장소(POI) 좌표는 개인 위치정보가 아니므로 신고 대상 아님 — 안내 자료에 **"맛집 검색(가게 위치만 제공)"이 신고 불필요 예시로 명시**되어 있음.

## 2. 맛담 실사 결과 (2026-07-17 코드 기준)

| 위치 사용처 | 처리 방식 | 서버 전송 |
|---|---|---|
| 가까운순 정렬 (`app/discover.tsx`) | `lib/geo.ts`로 기기 위치 취득 → 컴포넌트 state(`userLoc`)에서 클라이언트 정렬만 수행 | ❌ 없음 |
| 지도 내 위치 표시 (`components/RestaurantMap.tsx` / `.native.tsx`) | 웹: Leaflet 로컬 마커 / 네이티브: OS `showsUserLocation` 점 표시 | ❌ 없음 |
| 역지오코딩 (`api/extract-place.js` → Nominatim) | **가게 좌표**(붙여넣은 지도 링크에서 추출)만 변환 — 이용자 위치 아님 | 해당 없음 (POI) |
| 맛집 좌표 저장 (Supabase) | 가게(POI) 좌표 — 개인 위치정보 아님 | 해당 없음 (POI) |

→ 이용자 개인 위치정보가 맛담 서버·제3자로 전송되는 경로가 **없다**. 개인정보처리방침 3조("기기 위치는 서버에 저장하지 않음")와 실제 구현이 일치.

## 3. 결론

- **현재 기능 범위에서는 위치기반서비스사업 신고 불필요.**
- 근거: 개인 위치정보를 사업자가 수집하지 않고 단말기 내 처리만 함 + POI 좌표는 개인 위치정보가 아님.

## 4. 재검토 트리거 — 아래 기능을 추가하는 순간 신고 필요

1. 가까운순·주변 검색을 **서버 측**에서 처리 (이용자 좌표를 API로 전송)
2. 실시간 위치 공유, 친구 위치 보기 등
3. 이용자 위치 기반 푸시(지오펜싱)
4. 이용자 위치 로그 수집·분석 (분석 도구에 GPS 좌표 이벤트 속성으로 넣는 것 포함 — **PostHog 이벤트에 좌표 넣지 말 것**)

## 5. 그때의 절차 (소상공인 특례)

소상공인·1인창조기업은 **신고 없이 사업 개시 가능**, 개시 후 **1개월 이내에** 전자민원센터(emsit.go.kr)에 "소상공인등의 위치기반서비스사업 신고"를 하면 된다. 무료, 처리 최대 2주. 제출물: 사업내용 설명자료 + 사업자등록증 + 소상공인확인서(중소기업현황정보시스템 발급).
